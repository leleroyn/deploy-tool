# Deploy Tool Web 启动说明

## 环境要求

WSL（Ubuntu-24.04）中需要有 Node.js 18+，如果没有请先安装：

```bash
# 安装 Node.js 20（只需做一次）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node --version
npm --version
```

---

## 项目路径（WSL 内访问 Windows 文件）

Windows 路径 `f:\软件项目\deploy-tool` 对应 WSL 路径：

```
/mnt/f/软件项目/deploy-tool
```

---

## 访问密码配置

启动后端前，通过环境变量设置访问密码（不设置默认为 `admin123`）：

```bash
export DEPLOY_PASSWORD=your_password
```

建议加到 `~/.bashrc` 里持久化：

```bash
echo 'export DEPLOY_PASSWORD=your_password' >> ~/.bashrc
source ~/.bashrc
```

---

## 启动步骤（必须严格按顺序）

### 第一步：加载 SSH 密钥（每次重开终端都要做）

> **关键：必须在启动后端的同一个终端里执行，否则后端无法继承 SSH_AUTH_SOCK 变量，脚本调用会报错退出码 255。**

```bash
# 启动 ssh-agent 并加载私钥（会提示输入一次密码）
eval $(ssh-agent -s)
ssh-add /home/ubuntu/.ssh/id_rsa

# 验证密钥已加载
ssh-add -l
# 应显示类似：4096 SHA256:xxxx /home/ubuntu/.ssh/id_rsa (RSA)
```

### 第二步：启动后端（端口 3001）

**在同一个终端**（即刚才执行 ssh-add 的终端）继续执行：

```bash
cd /mnt/f/软件项目/deploy-tool/server

# 首次运行需安装依赖（只需做一次）
npm install

# 启动后端
npm run dev
```

看到以下输出说明后端成功：

```
[INFO] ts-node-dev ver. 2.0.0 ...
Deploy Tool Server running on http://localhost:3001
```

### 第三步：启动前端（端口 5173）

**另开一个新的 WSL 终端**，执行：

```bash
cd /mnt/f/软件项目/deploy-tool/web

# 首次运行需安装依赖（只需做一次）
npm install

# 启动前端
npm run dev
```

看到以下输出说明前端成功：

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

### 第四步：浏览器访问

两个服务都启动后，在 Windows 浏览器中打开：

```
http://localhost:5173
```

---

## 完整启动命令（快速参考）

```bash
# === 终端一（后端，含 SSH agent）===
eval $(ssh-agent -s)
ssh-add /home/ubuntu/.ssh/id_rsa
cd /mnt/f/软件项目/deploy-tool/server && npm run dev

# === 终端二（前端）===
cd /mnt/f/软件项目/deploy-tool/web && npm run dev
```

## 常见问题

### 执行操作退出码 255 / 无响应

原因：SSH agent 未在后端进程的环境变量中生效。

解决：停掉后端，在同一个终端重新按顺序执行 `eval $(ssh-agent -s)` → `ssh-add` → `npm run dev`。

### 前端报 `ECONNREFUSED 127.0.0.1:3001`

原因：后端没有启动，或后端启动在不同终端导致端口未监听。

解决：检查后端终端是否显示 `Deploy Tool Server running on http://localhost:3001`。

### 端口被占用

```bash
# 查看占用端口的进程
lsof -i :3001
lsof -i :5173

# 强制释放
kill -9 $(lsof -ti :3001)
kill -9 $(lsof -ti :5173)
```

### 依赖安装失败

```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

## 自动加载 SSH 密钥（可选配置）

如果每次输密码太麻烦，可以把以下内容加到 `~/.bashrc`，这样每次打开终端自动加载：

```bash
# 自动启动 ssh-agent 并加载密钥（加到 ~/.bashrc 末尾）
if [ -z "$SSH_AUTH_SOCK" ]; then
    eval $(ssh-agent -s) > /dev/null
fi
ssh-add -l &>/dev/null || ssh-add /home/ubuntu/.ssh/id_rsa
```

执行 `source ~/.bashrc` 生效。

## 生产环境部署（Docker 方式）

> 推荐在**服务器上长期运行**时使用这个方式。容器会自动重启，不需要保持终端开着，也不需要在服务器上安装 Node.js。

### 前提条件

服务器上需要安装 Docker 和 Docker Compose。如果没有，执行以下命令安装（Ubuntu/Debian）：

```bash
# 一键安装 Docker
curl -fsSL https://get.docker.com | sh

# 把当前用户加入 docker 组（这样不用每次都 sudo）
sudo usermod -aG docker $USER

# 重新登录一次，让 docker 组权限生效
exit
# 重新 ssh 连接服务器...

# 验证安装成功
docker --version
docker compose version
```

---

### 第一步：把项目文件上传到服务器

把整个 `deploy-tool` 文件夹上传到服务器，例如放在 `/opt/deploy-tool`：

```bash
# 在本机执行（把 your-server-ip 替换成服务器 IP）
scp -r /path/to/deploy-tool root@your-server-ip:/opt/deploy-tool
```

或者用 git clone，或者手动通过 SFTP 工具（如 FileZilla）上传都可以。

**上传完后，确认服务器上有以下文件：**

```
/opt/deploy-tool/
├── Dockerfile
├── docker-compose.yml
├── .dockerignore
├── script/
│   ├── config.ini      ← 要配置好服务器信息
│   ├── deploy.sh
│   ├── backup_pj.sh
│   └── check_ports.sh
├── server/
└── web/
```

---

### 第二步：修改密码

编辑 `docker-compose.yml`，把默认密码 `admin123` 改成你自己的密码：

```bash
cd /opt/deploy-tool
nano docker-compose.yml
```

找到这一行改掉：

```yaml
- DEPLOY_PASSWORD=admin123   # ← 改成你的密码
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

---

### 第三步：配置 SSH 密钥

容器需要 SSH 密钥才能连接到你的目标服务器。

**检查服务器上有没有 SSH 密钥：**

```bash
ls ~/.ssh/id_rsa
```

如果没有，生成一个：

```bash
ssh-keygen -t rsa -b 4096
# 一路回车即可
```

**把公钥复制到目标服务器（让脚本可以免密登录）：**

```bash
# 替换成你要部署的目标服务器 IP 和用户名
ssh-copy-id root@192.168.1.100
```

docker-compose.yml 里已经配好了密钥挂载：

```yaml
- ~/.ssh:/home/deploy/.ssh:ro
```

这行会把服务器的 `~/.ssh` 目录只读挂载进容器，容器里的脚本执行 SSH 时就能找到密钥。

---

### 第四步：构建并启动容器

```bash
cd /opt/deploy-tool

# 构建镜像并启动（首次需要几分钟下载依赖）
docker compose up -d --build
```

看到以下输出说明成功：

```
[+] Building ... 
[+] Running 1/1
 ✔ Container deploy-tool  Started
```

**在浏览器里打开：**

```
http://服务器IP:3001
```

输入你设置的密码，就能看到管理界面了。

---

### 日常管理命令

```bash
# 查看运行状态
docker compose ps

# 查看实时日志（排查问题时用）
docker compose logs -f

# 停止服务
docker compose stop

# 重新启动
docker compose start

# 更新代码后重新构建部署（会有短暂停机）
docker compose up -d --build

# 彻底删除容器（不影响 script/ 和 config.ini，数据不会丢）
docker compose down
```

---

### 修改配置后如何生效

`script/config.ini` 是直接挂载进容器的，**改完文件不需要重新构建**，直接重启容器即可：

```bash
docker compose restart
```

### 常见问题

**容器启动了但网页打不开**

```bash
# 检查容器是否真的在运行
docker compose ps

# 检查端口是否监听
ss -tlnp | grep 3001

# 检查防火墙是否放行了 3001 端口
# Ubuntu 用 ufw：
sudo ufw allow 3001
```

**脚本执行报错 Permission denied**

```bash
# 检查脚本是否有执行权限
ls -la /opt/deploy-tool/script/

# 加上执行权限
chmod +x /opt/deploy-tool/script/*.sh
```

**SSH 连接目标服务器失败（退出码 255）**

原因：容器内找不到 SSH 密钥，或密钥权限不对。

```bash
# 检查宿主机密钥是否存在
ls -la ~/.ssh/id_rsa

# 确认密钥权限正确（必须是 600）
chmod 600 ~/.ssh/id_rsa
chmod 700 ~/.ssh

# 重启容器让挂载生效
docker compose restart
```

**想换一个端口**

编辑 `docker-compose.yml`，把 `"3001:3001"` 改成 `"你想要的端口:3001"`，例如改成 8080：

```yaml
ports:
  - "8080:3001"
```

然后 `docker compose up -d --build` 重启，访问 `http://服务器IP:8080`。
