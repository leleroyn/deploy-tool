# Deploy Tool Web 启动说明

## 环境要求

WSL（Ubuntu-24.04）中需要有 Node.js 18+，如果没有请先安装：

```bash
# 安装 Node.js 20（只需做一次）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node --version
npm --version
```

---

## 项目路径（WSL 内访问 Windows 文件）

Windows 路径 `f:\软件项目\deploy-tool` 对应 WSL 路径：

```
/mnt/f/软件项目/deploy-tool
```

---

## 访问密码配置

启动后端前，通过环境变量设置访问密码（不设置默认为 `admin123`）：

```bash
export DEPLOY_PASSWORD=your_password
```

建议加到 `~/.bashrc` 里持久化：

```bash
echo 'export DEPLOY_PASSWORD=your_password' >> ~/.bashrc
source ~/.bashrc
```

---

## 启动步骤（必须严格按顺序）

### 第一步：加载 SSH 密钥（每次重开终端都要做）

> **关键：必须在启动后端的同一个终端里执行，否则后端无法继承 SSH_AUTH_SOCK 变量，脚本调用会报错退出码 255。**

```bash
# 启动 ssh-agent 并加载私钥（会提示输入一次密码）
eval $(ssh-agent -s)
ssh-add /home/ubuntu/.ssh/id_rsa

# 验证密钥已加载
ssh-add -l
# 应显示类似：4096 SHA256:xxxx /home/ubuntu/.ssh/id_rsa (RSA)
```

### 第二步：启动后端（端口 3001）

**在同一个终端**（即刚才执行 ssh-add 的终端）继续执行：

```bash
cd /mnt/f/软件项目/deploy-tool/server

# 首次运行需安装依赖（只需做一次）
npm install

# 启动后端
npm run dev
```

看到以下输出说明后端成功：

```
[INFO] ts-node-dev ver. 2.0.0 ...
Deploy Tool Server running on http://localhost:3001
```

### 第三步：启动前端（端口 5173）

**另开一个新的 WSL 终端**，执行：

```bash
cd /mnt/f/软件项目/deploy-tool/web

# 首次运行需安装依赖（只需做一次）
npm install

# 启动前端
npm run dev
```

看到以下输出说明前端成功：

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

### 第四步：浏览器访问

两个服务都启动后，在 Windows 浏览器中打开：

```
http://localhost:5173
```

## 完整启动命令（快速参考）

```bash
# === 终端一（后端，含 SSH agent）===
eval $(ssh-agent -s)
ssh-add /home/ubuntu/.ssh/id_rsa
cd /mnt/f/软件项目/deploy-tool/server && npm run dev

# === 终端二（前端）===
cd /mnt/f/软件项目/deploy-tool/web && npm run dev
```

## 常见问题

### 执行操作退出码 255 / 无响应

原因：SSH agent 未在后端进程的环境变量中生效。

解决：停掉后端，在同一个终端重新按顺序执行 `eval $(ssh-agent -s)` → `ssh-add` → `npm run dev`。

### 前端报 `ECONNREFUSED 127.0.0.1:3001`

原因：后端没有启动，或后端启动在不同终端导致端口未监听。

解决：检查后端终端是否显示 `Deploy Tool Server running on http://localhost:3001`。

### 端口被占用

```bash
# 查看占用端口的进程
lsof -i :3001
lsof -i :5173

# 强制释放
kill -9 $(lsof -ti :3001)
kill -9 $(lsof -ti :5173)
```

### 依赖安装失败

```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

## 自动加载 SSH 密钥（可选配置）

如果每次输密码太麻烦，可以把以下内容加到 `~/.bashrc`，这样每次打开终端自动加载：

```bash
# 自动启动 ssh-agent 并加载密钥（加到 ~/.bashrc 末尾）
if [ -z "$SSH_AUTH_SOCK" ]; then
    eval $(ssh-agent -s) > /dev/null
fi
ssh-add -l &>/dev/null || ssh-add /home/ubuntu/.ssh/id_rsa
```

执行 `source ~/.bashrc` 生效。

## 生产环境部署（Docker 方式）

> 推荐在**服务器上长期运行**时使用这个方式。容器会自动重启，不需要保持终端开着，也不需要在服务器上安装 Node.js。

### 前提条件

服务器上需要安装 Docker。如果没有，执行以下命令安装（Ubuntu/Debian）：

```bash
# 一键安装 Docker
curl -fsSL https://get.docker.com | sh

# 验证安装成功
docker --version
```

---

### 第一步：准备项目文件

把 `deploy-tool` 文件夹上传到服务器，例如放在 `/root/deploy-tool`（或任意目录）：

```bash
# 在本机执行（把 your-server-ip 替换成服务器 IP）
scp -r /path/to/deploy-tool root@your-server-ip:/root/deploy-tool
```

也可以用 git clone 或 SFTP 工具（如 FileZilla）上传。

**确认服务器上有以下文件：**

```
/root/deploy-tool/
├── script/
│   ├── config.ini      ← 要配置好目标服务器信息
│   ├── deploy.sh
│   ├── backup_pj.sh
│   └── check_ports.sh
└── logs/               ← 手动创建（日志目录）
```

---

### 第二步：修改 config.ini

编辑 `script/config.ini`，确认 SSH 配置中的密钥路径指向服务器上的实际路径：

```bash
nano /root/deploy-tool/script/config.ini
```

SSH 段示例（以 root 用户运行，密钥路径为 `/root/.ssh/id_rsa`）：

```ini
[ssh]
user = root
key = /root/.ssh/id_rsa
```

---

### 第三步：配置 SSH 密钥

容器需要 SSH 密钥才能连接到目标服务器。

**检查密钥是否存在：**

```bash
ls ~/.ssh/id_rsa
```

如果没有，生成一个：

```bash
ssh-keygen -t rsa -b 4096
# 一路回车即可
```

**把公钥复制到目标服务器：**

```bash
# 替换成你要部署的目标服务器 IP
ssh-copy-id root@192.168.1.100
```

---

### 第四步：创建日志目录并启动容器

```bash
cd /root/deploy-tool

# 创建日志目录（必须提前创建，避免 docker 自动创建时 owner 是 root）
mkdir -p logs

# 启动容器
docker run -d \
  --name deploy-tool \
  --restart unless-stopped \
  -p 3001:3001 \
  -u root \
  -e DEPLOY_PASSWORD=你的密码 \
  -e SCRIPT_DIR=/app/script \
  -e CONFIG_FILE=/app/config.ini \
  -e SKIP_SSH_INIT=1 \
  -e LOG_BASE_DIR=/app/logs \
  -v "$(pwd)/script/config.ini:/app/config.ini:ro" \
  -v "$(pwd)/script:/app/script:ro" \
  -v "/root/.ssh:/root/.ssh:ro" \
  -v "$(pwd)/logs:/app/logs" \
  leleroyn/deploy-tool:0.1
```

> 把 `你的密码` 替换成实际密码。

**在浏览器里打开：**

```
http://服务器IP:3001
```

输入密码，就能看到管理界面了。

---

### 日常管理命令

```bash
# 查看运行状态
docker ps -f name=deploy-tool

# 查看实时日志（排查问题时用）
docker logs -f deploy-tool

# 停止容器
docker stop deploy-tool

# 重新启动
docker start deploy-tool

# 删除并重建容器（更新镜像或修改环境变量后用）
docker stop deploy-tool && docker rm deploy-tool
# 然后重新执行上面的 docker run 命令

# 强制拉取最新镜像后重建
docker pull leleroyn/deploy-tool:0.1
docker stop deploy-tool && docker rm deploy-tool
# 重新执行 docker run 命令
```

---

### 修改配置后如何生效

`script/config.ini` 是直接挂载进容器的，**改完文件立即生效**，不需要重启容器，下次执行脚本时自动读取最新配置。

如果修改了**环境变量**（如密码），需要删除并重建容器：

```bash
docker stop deploy-tool && docker rm deploy-tool
# 重新执行 docker run，带上新的 -e DEPLOY_PASSWORD=新密码
```

### 常见问题

**容器启动了但网页打不开**

```bash
# 检查容器是否在运行
docker ps -f name=deploy-tool

# 检查端口是否监听
ss -tlnp | grep 3001

# 检查防火墙是否放行了 3001 端口
sudo ufw allow 3001
```

**脚本执行报错 Permission denied**

```bash
# 检查脚本是否有执行权限
ls -la /root/deploy-tool/script/

# 加上执行权限
chmod +x /root/deploy-tool/script/*.sh
```

**SSH 连接目标服务器失败（退出码 255）**

原因：SSH 密钥路径不对，或密钥权限不正确。

```bash
# 确认密钥存在且权限正确
ls -la ~/.ssh/id_rsa
chmod 600 ~/.ssh/id_rsa
chmod 700 ~/.ssh

# 确认 config.ini 里 key 的路径是 /root/.ssh/id_rsa
grep key /root/deploy-tool/script/config.ini
```

**想换一个端口**

重建容器时，把 `-p 3001:3001` 改成 `-p 你想要的端口:3001`，例如改成 8080：

```bash
docker stop deploy-tool && docker rm deploy-tool

docker run -d \
  --name deploy-tool \
  ...
  -p 8080:3001 \
  ...
```

然后访问 `http://服务器IP:8080`。
